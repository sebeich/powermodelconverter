Tool Capabilities

cgmes

Native pandapower CGMES/CIM import is wrapped for balanced models, and balanced export is validated by re-importing the generated CGMES package through pandapower. Current export scope is the supported bus-branch subset with one slack source, lines, and constant-power loads.

  • Balanced: import=yes, export=yes, validation=yes
  • Unbalanced: import=no, export=no, validation=no

matpower

Balanced MATPOWER cases are validated. Unbalanced semantics are outside the MATPOWER exchange contract.

  • Balanced: import=yes, export=yes, validation=yes
  • Unbalanced: import=no, export=no, validation=no

opendss

OpenDSS is validated as both an import and export backend for the current balanced and starter three-phase distribution subsets.

  • Balanced: import=yes, export=yes, validation=yes
  • Unbalanced: import=yes, export=yes, validation=yes

pandapower

Balanced JSON and native pandapower 3ph models are supported. Unbalanced validation currently targets native pandapower roundtrips.

  • Balanced: import=yes, export=yes, validation=yes
  • Unbalanced: import=yes, export=yes, validation=yes

powermodels

Balanced PowerModels JSON export is validated with Julia AC power flow.

  • Balanced: import=no, export=yes, validation=yes
  • Unbalanced: import=no, export=no, validation=no

powermodelsdistribution

Validated for the current DSS-compatible unbalanced subset. Import uses the OpenDSS parser on PowerModelsDistribution-compatible DSS inputs, and export/validation are signed off for native asymmetric-load feeders within the current supported subset.

  • Balanced: import=no, export=no, validation=no
  • Unbalanced: import=yes, export=yes, validation=yes

pypower

Planned via the MATPOWER-style balanced exchange contract.

  • Balanced: import=no, export=no, validation=no
  • Unbalanced: import=no, export=no, validation=no

pypsa

Balanced AC PyPSA import/export is validated for the current line-based transmission subset against pandapower and PyPSA power flow. Current scope excludes unbalanced networks and does not yet sign off transformer-, shunt-, link-, store-, or storage-unit-heavy PyPSA models.

  • Balanced: import=yes, export=yes, validation=yes
  • Unbalanced: import=no, export=no, validation=no
